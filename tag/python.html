Moving to a static site generator

Published: Thu 09 April 2015
By Sudhanshu Mishra

In Programming.

tags: Python

Finally I moved to a static site generator for my homepage + blog. Previously, I was using blogger which was really annoying when writing a technical post. Code formatting was bizarre!

Of all the static website generators I chose Pelican. It is written in Python and it doesn’t require any database at the server side. I can write articles in markdown which gives me full control over code highlighting! I can also write math using MathJax.

$$\lim_{x \to 0^+}\left(\frac{1}{x} \sin{\left (x \right )}\right) + \int_{0}^{\infty} e^{x}\, dx$$

I know that these things are also possible with Jekyll but the fact that Jekyll builds sometime take minutes, set me off. Nevertheless, Pelican is in Python!

Pelican also gives you a nice way of importing contents from your old blog. I used following command to do that.

$ pelican-import  --feed http://blog.sudhanshumishra.in/feeds/posts/default/ -o content/ -m markdown

This reads the stream of RSS feed and converts them into markdown files. I’ll explain what it actually did a bit later.

Install

You can install it very easily with pip. For a full installation(with markdown support) issue following command.

$ pip install --upgrade pelican markdown typogriphy

Kickstart your website

Now create your project folder and cd into it. After navigating into your project folder issue following command.

$ pelican-quickstart

If Pelican was installed successfully on your computer, it will prompt you with a questionnaire having very simple questions. Answer them and it’ll generate skeleton files for your website.

The directory structure looks somewhat like this:

yourproject/
├── content
│   └── (pages)
├── output
├── develop_server.sh
├── fabfile.py
├── Makefile
├── pelicanconf.py       # Main settings file
└── publishconf.py       # Settings to use when ready to publish

Add contents

Adding content is very easy. You just have to create markdown files with extension .md in the content directory and it will automatically convert them into publishable html. The command to import contents from blogger actually converted all my posts into markdown files and saved them into the content directory.

Pelican follows a certain format for articles. For instance, here’s the format for this article:

Title: Moving to a static site generator
Date: 2015-04-09 06:50
Author: Sudhanshu Mishra
Category: Programming
Tags: Python
Slug: moving-to-a-static-site-generator

Finally I moved to a static site generator for my homepage + blog website.
Previously, I was using [blogger](http://blog.sudhanshumishra.in) which was really
annoying while writing a technical post. Code formatting was bizarre!

...

Publish

Once you are done with the article just issue following command to generate html.

$ make html

This will generate html along with assets in the output directory. Since I’m using GitHub pages to host my website, I used output directory as my git repository having CNAME file. Now just commit those files and push to your repository(your_username.github.io). Learn more about GitHub pages here.

If you want to change theme or add plugins, visit the documentation.
